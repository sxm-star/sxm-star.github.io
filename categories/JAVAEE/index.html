tomcat的3种运行模式

tomcat的运行模式有3种：
1、bio 默认的模式,性能非常低下,没有经过任何优化处理和支持.
2、nio 利用java的异步io护理技术,no blocking IO技术.
3、apr 安装起来最困难,但是从操作系统级别来解决异步的IO问题,大幅度的提高性能.

启动nio模式

修改server.xml里的Connector节点,修改protocol为org.apache.coyote.http11.Http11NioProtocol

JSP基础语法--JSP内置对象response对象

response对象

response对象的主要作用是用于对客户端的请求进行回应，将WEB服务器处理后的结果发回给客户端。response对象属于javax.servlet.http.HttpServletResponse接口的实例。

response对象的常用方法

设置刷新头信息

直接使用setHeader()方法，将头信息名称设置为refresh,同时指定刷新的时间。

  • 示例
    <%@ page contentType="text/html" pageEncoding="GBK"%>
<html>
<head><title>请求头信息</title></head>
<body>
<%!	// 定义全局变量
  int count = 0 ;
%>
<%
  response.setHeader("refresh","2") ;	// 页面2秒一刷新
%>
<h3>已经访问了<%=count++%>次！</h3>
</body>
</html>
  • 刷新并跳转别的页面
    <%@ page contentType="text/html" pageEncoding="GBK"%>
<html>
<head><title>刷新并跳转别的页面</title></head>
<body>
<h3>3秒后跳转到hello.htm页面，如果没有跳转请按<a href="hello.htm">这里</a>！</h3>
<%
  response.setHeader("refresh","3;URL=hello.htm") ;	// 3秒后跳转到hello.htm页面
%>
</body>
</html>

    刷新的HTML指令

    <META HTTP-EQUIV="refresh" CONTENT="3;URL=hello.htm" />
    示例

    页面跳转

    在JSP中除了可以通过头信息的方式完成跳转，也可以使用response对象的sendRedirect()方法直接完成页面的跳转。

    两种跳转的区别

  1. jsp:forward属于服务器跳转，跳转之后地址栏的信息并不会有任何的改变，而response.sendRedirect()属于客户端跳转，跳转之后地址栏会改变，
  2. 在使用request范围属性的时候，只有服务器跳转才能将request属性保存到跳转页，而如果是客户端跳转，则无法进行属性的传递。
  3. 如果使用的是服务器端跳转的话，则执行到跳转语句之后会立刻进行跳转，如果使用的客户端跳转，则是在整个页面执行完之后才能执行跳转。

JSP基础语法--JSP九大内置对象总览

内置对象

在JSP中为了简化用户的开发，提供了九个内置对象，这些内置对象将由容器为用户进行实例化，而用户直接使用即可，而不用像在JAVA中那样，必须通过关键字new进行实例化对象之后才可以使用。

JSP九大内置对象及其对应类型

四中属性范围

1. 在JSP中提供了四中属性的保存范围，所谓的属性保存范围，指的是一个设置的对象，可以在多少个页面中保存并可以继续使用
2. 四中属性范围:
  1) pageContext:只在一个页面中保存属性，跳转之后无效。
  2) request：只在一次请求中保存，服务器跳转后依然有效，客户端跳转无效。
  3) session: 在一次会话范围中，无论何种跳转都可以使用，但是新开浏览器无法使用。
  4) application: 在整个服务器上保存，所有用户都可以使用

属性操作方法

JSP基础语法--JSP的forward跳转指令

跳转的作用

  • 从WEB中可以使用jsp:forward指令，将一个用户的请求(request),从一个页面传递到另一个页面
  • 页面跳转语法:
    • 不传递参数:
        <jsp:forward page="{要包含的文件路径 | <%=表达式%>}" />
    • 传递参数：
        <jsp:forward page="{要包含的文件路径 | <%=表达式%>}" >
      <jsp:param name="参数名称" value=“参数内容” />
      ... 可以向被跳转页面中传递多个参数
  </jsp:forward>

      示例

      forward_demo02.jsp示例：
      <html>
<head><title>跳转传参示例</title></head>
<body>
<%
String username = "xuanming" ;
%>
<jsp:forward page="forward_demo02.jsp">
<jsp:param name="name" value="<%=username%>"/>
<jsp:param name="info" value="learning"/>
</jsp:forward>
</body>
</html>
      forward_demo02.jsp 示例：
      <%@ page contentType="text/html" pageEncoding="GBK"%>

      小结

  • 使用跳转可以完成页面请求的传递。
  • 跳转操作属于服务器端跳转，跳转之后的页面路径不改变。

JSP基础语法--JSP的include包含指令

一般像以下的页面结构，都会采用到包含指令

静态包含：

静态包含语法：
<%@include file="包含文件路径" %>
  • 定义要包含的三个页面
    info.htm 文件内容：
<h2><font color="red">this is info.htm</font>

    info.jsp 文件内容：
<h2><font color="red"> <%="this is info.jsp"%> </font>

    info.inc 文件内容：
<h2><font color="red"> info.inc</font>
  • 使用静态指令包含
    include_demo示例：
    <%@ page contentType="text/html" pageEncoding="UTF-8" %>
<html>
<head>
 <title>静态包含demo</title>

</body>
</html>
  • 静态包含的处理流程

动态包含

动态包含<jsp:include>
  • 情况一：被包含的页面属于静态文件，那么就像<%@include%>一样直接包含进来
  • 情况二： 被包含的页面属于动态页面，则要分别处理后再进行包含。

    动态包含的语法定义

  • 定义一：包含的时候不传参数。
    <jsp:include page="路径"/>
  • 定义二：包含的时候传递参数。
    <jsp:include page="路径">
 <jsp:param name="参数名称" value="参数内容"/>
 <jsp:param name="参数名称" value="参数内容"/>
 <jsp:param name="参数名称" value="参数内容"/>
 <jsp:param name="参数名称" value="参数内容"/>
</jsp:include>
    所有接收的参数方式使用 request.getParameter()完成。
  • 完成与静态包含类似的功能
    <%@page pageEncoding="UTF-8" %>
<jsp:include page="info.htm" />
<jsp:include page="info.jsp" />
<jsp:include page="info.inc" />
    这一点上包含的内容和静态包含是没有区别的
  • 动态包含传递参数
    param.jsp 示例：
    <%@ page contentType="text/html" pageEncoding="UTF-8"%>
<%
     String userName = "xuanming";
%>
<h1>动态包含并传递参数</h1>
<jsp:include page="receive_param.jsp">
    <jsp:param name="name" value="<%=userName %>" />
    <jsp:param name="info" value="learnning " />
</jsp:include>
    receive_param.jsp 示例：
    <%@ page contentType="text/html" pageEncoding="UTF-8" %>
<h1>参数一：<%=request.getParameter("name")%></h1>
<h1>参数二：<%=request.getParameter("info")%></h1>

总结

  • 静态包含：
  • 语法:
       <%@include file="路径" %>
  • 处理流程：先包含后一起进行处理
  • 动态包含:
  • 语法:
     <jsp:include page="路径" />
  • 处理流程：如果包含的是静态页面则将只是将文件内容导入进来，如果包含的是动态文件，则将处理后的结果包含进来（先处理后包含）
  • 以后开发尽可能用动态包含，可以避免局部变量同名冲突。
