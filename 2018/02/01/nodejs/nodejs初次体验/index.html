nodejs初次体验

node安装介绍(OSX环境下)

  1. 通过brew instsll node 安装
  2. 通过官网 https://nodejs.org/en/ 下载安装包，一路next 安装 ，默认 node、npm命令在 /usr/local/bin/
    查看安装版本:  node -v
执行node计算:  node 回车  3+5  回车  就可看到结果  8
退出node ：press ^C again or type .exit

node环境下web框架 express

  1. express官网地址: https://www.npmjs.com/package/express
  2. express框架安装: sudo npm install -g express-generator@4
  3. 创建web项目: mkdir pro_name && cd pro_name && express -e service
  4. 查看项目依赖的框架: cd service && cat package.json
  5. 安装依赖的框架: npm install
  6. 在service目录下会多出 node_modules 目录
  7. 启动项目: npm start, 默认3000端口,查看web启动情况：http://localhost:3000/
  8. 可以修改端口: cd service/bin && cat www && 修改端口就可以了

项目运行流程介绍

  1. webstorm打开 service,app.js是启动入口,里面有路由信息
    var index = require(‘./routes/index’);
    var users = require(‘./routes/users’);
  2. 路由控制跳转 或者渲染情况
    2.1. 在route目录下有路由文件，index.js 是指把 json数据 title传给 view目录下的 index.ejs进行渲染,再返回给客户端
    router.get(‘/‘, function(req, res, next) {
    res.render(‘index’, { title: ‘Express’ });
    });
    2.2 在route目录下有路由文件，users.js 是直接返回数据给客户端，不进行渲染
    router.get(‘/‘, function(req, res, next) {
    res.send(‘respond with a resource’);
    });
  3. 安装一个依赖包，自动监听修改重启操作: cd service && sudo npm install -g supervisor &&
  4. 通过supervisor启动项目: supervisor bin/www

demo 接口服务示例



service的public目录下 建立data目录

  1. data目录下 建 json文件 ，例如 it.json
  2. 在 routes目录新建 data.js 做具体URL控制 和 逻辑判断处理
  3. 最后在 app.js绑定 新建的 data 路由
    var data = require(‘./routes/data’);
    app.use(‘/data’,data);

  3. var str = xss(name);
